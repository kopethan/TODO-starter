# Public UI implementation plan by slices

## Slice 1 — Theme + public shell
- Simplify the public palette to white/black neutrals with one blue accent.
- Tighten cards, buttons, inputs, and header chrome so the public product feels cleaner and more premium.
- Make public search a first-class action in the shell.

## Slice 2 — Home page becomes a real feed
- Replace the oversized intro feeling with a search-first hero.
- Promote high-signal reports as the main reading surface.
- Add a calmer “Read before you act” entity block.
- Introduce simple pattern cards so the page feels like a trust-reading product instead of a generic homepage.

## Slice 3 — Reports archive
- Rebuild the top controls around strong search + practical filters.
- Improve report-card hierarchy so severity, verification, entity, and date scan faster.
- Add sorting and contextual guidance so reports stay clearly separate from canonical reference content.

## Slice 4 — Entity page polish
- Strengthen the entity header with a “What to know first” summary.
- Keep the three-column desktop structure, but make section hierarchy clearer.
- Improve related reports so they feel connected evidence, not appended content.
- Add lighter mobile jump navigation.

## Slice 5 — Validation
- Run typecheck after public UI updates.
- Keep the changes isolated to the public surface and shared UI primitives used by that surface.
