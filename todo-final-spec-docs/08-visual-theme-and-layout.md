# Visual Theme and Layout Direction

## Main direction

The preferred direction is:
- black and white as the main UI language
- one intentional blue accent
- flat design only
- minimal, serious, editorial rhythm

## Why

This better fits TODO than a colorful or noisy interface.

Desired feeling:
- clean
- calm
- precise
- confident
- not playful
- not dashboard-loud

## Color model

### Primary base
- black
- white
- grayscale neutrals

### Accent
Use one strong but controlled blue for:
- dividers
- active icons
- links
- buttons
- headers, where appropriate
- important emphasis

Use color rarely and intentionally.

## Divider-based layout

Recommended pattern:

```text
[header]
---divider---
[content block]
---divider---
[content block]
---divider---
[content block]
```

The divider should:
- create rhythm
- separate sections clearly
- feel attached to the layout
- adapt responsively
- not feel decorative or flashy

Use dividers mainly between major sections or sibling items, not inside every content card.

## Divider styling notes

Recommended:
- thin but visible
- blue-accented
- responsive width
- subtle, not glowing
- can extend out from navigational controls or tab rows when helpful

## Header / toolbar

If the bottom tool dock is used:
- keep the header clean
- avoid a permanently expanded search bar at the top
- keep top-level chrome minimal
- do not let Reports / Signals dominate primary public navigation

## Entity page layout

Recommended:
- centered, but wider than a narrow article column
- tabs used as the main structural switcher
- moderate horizontal expansion on desktop
- stacked sections on mobile
- avoid a "panel inside panel" feeling where possible

## Buttons and icons

Recommended:
- flat
- minimal
- consistent blue active state
- restrained radius
- no glossy or 3D styling

## Images

Use images when they help understanding:
- entity images for products/apps/websites/objects
- evidence images for reports
- avoid decorative filler imagery
